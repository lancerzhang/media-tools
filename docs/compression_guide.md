# 媒体压缩使用指南

## 概述

本工具提供两种方式进行媒体文件压缩：
1. Windows 批处理脚本（简单快速）
2. Python 脚本（功能更灵活）

## Windows 批处理方式

### 使用步骤

1. 确保已安装 FFmpeg 并添加到系统 PATH
2. 将 `scripts/compress_media.bat` 复制到要处理的目录
3. 双击运行脚本
4. 等待处理完成，查看统计信息

### 特点

- 自动递归处理所有子目录
- 支持的图片格式：JPG, PNG, BMP, TIFF, WebP
- 支持的视频格式：MP4, MKV, AVI, MOV, FLV, WMV, M4V
- 图片转换为 JPEG 格式（质量参数 q:v 4）
- 视频转换为 H.265/HEVC 编码（CRF 28）
- 自动跳过临时文件
- 显示处理进度和总耗时

### 自定义参数

编辑 `compress_media.bat` 文件：

```batch
:: 修改图片质量（2-31，值越小质量越高）
ffmpeg ... -q:v 4 ...

:: 修改视频质量（18-28，值越小质量越高）
ffmpeg ... -crf 28 ...
```

## Python 脚本方式

### 安装依赖

```bash
pip install -r requirements.txt
```

### 基本用法

```bash
# 压缩指定目录的所有媒体文件
python src/compress_media.py --input "your_folder"
```

### 高级选项

```bash
# 仅压缩图片
python src/compress_media.py --input "photos" --type image

# 仅压缩视频
python src/compress_media.py --input "videos" --type video

# 自定义视频质量（CRF 值）
python src/compress_media.py --input "media" --crf 26

# 自定义图片质量
python src/compress_media.py --input "media" --image-quality 3
```

### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--input` / `-i` | 要处理的文件夹路径 | 必填 |
| `--type` / `-t` | 处理类型：image/video/all | all |
| `--crf` | 视频压缩质量（18-28） | 28 |
| `--image-quality` | 图片压缩质量（2-31） | 4 |

## 质量参数建议

### 视频 CRF 值

| CRF | 质量 | 适用场景 |
|-----|------|----------|
| 18-20 | 极高质量 | 专业视频、需要后期处理 |
| 23-25 | 高质量 | 日常使用、平衡质量和大小 |
| 26-28 | 标准质量 | 4K/1080P 视频、节省空间 |

### 图片质量值

| 值 | 质量 | 说明 |
|----|------|------|
| 2-3 | 极高 | 接近无损 |
| 4-5 | 高 | 推荐日常使用 |
| 6-10 | 中等 | 明显压缩但可接受 |

## 注意事项

### ⚠️ 重要警告

1. **备份数据**：压缩操作会替换原文件，务必提前备份！
2. **不可逆**：压缩后无法恢复原始文件
3. **测试先行**：建议先用少量文件测试效果

### 💡 使用建议

1. **分批处理**：大量文件建议分批处理，避免长时间运行
2. **质量测试**：不同类型的视频可能需要不同的 CRF 值
3. **磁盘空间**：确保有足够的临时空间（至少原文件大小的 50%）
4. **处理时间**：
   - 图片：通常几秒到几十秒
   - 视频：取决于文件大小和质量设置，可能需要数分钟到数小时

### 🎯 最佳实践

1. **4K 视频**：使用 CRF 28，可节省 50-70% 空间
2. **1080P 视频**：使用 CRF 26-28
3. **720P 视频**：使用 CRF 24-26
4. **手机照片**：使用默认质量 4 即可
5. **专业照片**：使用质量 2-3

## 常见问题

### Q: 压缩后文件变大了？

A: 可能原因：
- 原文件已经是高度压缩的格式（如微信视频）
- 质量参数设置过高
- 建议：跳过已压缩的文件，或调整质量参数

### Q: 视频压缩很慢？

A: 正常现象，H.265 编码需要大量计算：
- CPU 性能影响很大
- 可以考虑使用硬件加速（需要修改脚本）
- 或者使用更快的编码器（质量会降低）

### Q: 如何使用 GPU 加速？

A: 修改 Python 脚本中的编码器：

```python
# 将 libx265 改为 hevc_videotoolbox (Mac) 或 hevc_nvenc (NVIDIA GPU)
'-c:v', 'hevc_videotoolbox',
# 注意：GPU 编码器不支持 CRF，需要使用比特率控制
'-b:v', '2000k',
```

### Q: 支持其他格式吗？

A: 可以通过修改脚本添加：
- 编辑 `IMAGE_EXTENSIONS` 或 `VIDEO_EXTENSIONS`
- 添加新的文件扩展名

## 技术细节

### 图片压缩

使用 FFmpeg 将各种格式转换为 JPEG：
```bash
ffmpeg -i input.png -q:v 4 output.jpg
```

### 视频压缩

使用 H.265/HEVC 编码器：
```bash
ffmpeg -i input.mp4 -c:v libx265 -crf 28 -tag:v hvc1 -c:a copy output.mp4
```

参数说明：
- `-c:v libx265`：使用 H.265 视频编码器
- `-crf 28`：恒定质量模式，值 28
- `-tag:v hvc1`：兼容性标签（适用于 Apple 设备）
- `-c:a copy`：音频直接复制，不重新编码
- `-map_metadata 0`：保留原始元数据
